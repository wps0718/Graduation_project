## 一、项目背景 【全局】

### 1.1 项目概述

「轻院二手」是一款面向**广东轻工职业技术大学**校内学生的微信小程序，旨在解决校园内闲置二手商品出售难的问题。平台参考闲鱼模式，结合校园特色，提供商品发布、浏览搜索、即时通讯、面交撮合等核心功能，让闲置物品在校园内高效流转。

### 1.2 项目背景与痛点

| 痛点 | 描述 |
|------|------|
| 信息分散 | 学生出售闲置物品依赖QQ群、微信群，信息零散、易刷屏、难检索 |
| 信任缺失 | 群内交易无信用体系，商品质量和卖家信誉无法保障 |
| 效率低下 | 买卖双方缺乏高效的沟通和交易管理工具 |
| 毕业浪费 | 每年毕业季大量物品被丢弃，缺乏集中的处理渠道 |

### 1.3 项目目标

| 目标 | 衡量指标 | 目标值 |
|------|----------|--------|
| 用户规模 | 注册认证用户数 | 上线3个月覆盖30%在校生 |
| 商品活跃度 | 日均新发布商品数 | ≥50件 |
| 交易转化 | 商品成交率 | ≥15% |
| 用户体验 | 平均交易完成时长 | ≤72小时 |

## 二、用户角色定义 【用户模块】

### 2.1 用户角色

| 角色 | 描述 | 权限 |
|------|------|------|
| **游客** | 未登录用户 | 浏览首页、搜索商品、查看商品详情 |
| **普通用户（未认证）** | 已登录但未完成校园认证 | 游客权限 + 收藏商品、发布商品（标注"未认证"）、聊天、下单 |
| **认证用户** | 已完成校园认证的学生 | 全部功能，展示"已认证"标识 |
| **管理员** | 后台管理人员 | 商品审核、举报处理、数据统计、运营管理 |

### 2.2 用户状态流转

```
游客 → [手机号/微信登录] → 普通用户（未认证）→ [提交校园认证] → 认证用户
                                                          ↓
                                                   [违规] → 封禁用户
                                                          ↓
                                                   [申请注销] → 注销中（30天）→ 永久删除
                                                                    ↓
                                                            [30天内恢复] → 正常用户
```

## 三、业务流程 【全局】

### 3.1 核心业务流程总览

```
│                        轻院二手 核心业务流程                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  卖家                          平台                    买家       │
│                                                                 │
│  [发布商品] ──────→ [人工审核] ──────→ [商品上架展示]              │
│                       │                    │                    │
│                    [驳回通知]          [浏览/搜索/收藏]            │
│                                            │                    │
│                                     [点击"我想要"]               │
│                                            │                    │
│  ←──────────── [进入IM聊天，协商价格/面交] ──────────→            │
│                                            │                    │
│  [修改价格（可选）]                  [买家确认购买]                │
│                                            │                    │
│                                     [生成订单-待面交]             │

│                                            │                    │

└─────────────────────────────────────────────────────────────────┘
```

### 3.2 用户注册/登录流程 【用户模块】

```
用户打开小程序
    │
    ├─→ [微信一键登录] → 获取openId → 自动创建账号 → 进入首页
    │
    ├─→ [手机号+密码登录] → 校验账号密码 → 进入首页
    │
    └─→ [短信验证登录] → 输入手机号 → 获取验证码 → 校验通过
                              │
                              └─→ 未注册手机号 → 自动创建账号 → 进入首页

    ※ 首次登录需勾选同意《用户协议》和《隐私政策》
    ※ 登录成功后返回JWT Token，后续请求携带Token
```

### 3.3 校园认证流程 【认证模块】

```
用户进入认证页面
    │
    ├─→ 填写：学院（下拉选择）、学号、班级
    ├─→ 上传：一卡通照片 或 3.0系统截图
    │
    └─→ [提交认证] → 后台人工审核
                        │
                        ├─→ [通过] → 用户标记"已认证" → 站内通知
                        └─→ [驳回] → 站内通知（附驳回原因）→ 用户可重新提交
```

### 3.4 商品发布流程 【商品模块】

```
用户点击底部Tab"发布"
    │
    ├─→ 上传图片（最多9张，支持拖拽排序，首张为封面）
    ├─→ 填写：商品名称、二手价格、原价、产品类型（分类）、磨损程度、描述
    ├─→ 选择：交易校区、面交地点（预设+自定义）
    │
    └─→ [发布] → 进入审核队列
                    │
                    ├─→ [审核通过] → 商品上架，首页可见 → 站内通知
                    └─→ [审核驳回] → 站内通知（附原因）→ 用户可编辑后重新提交
```

### 3.5 交易完整流程 【订单模块】

```
买家点击"我想要"
    │
    └─→ 跳转至与卖家的IM聊天界面（聊天中关联商品卡片）
            │
            └─→ 双方协商价格、面交时间地点
                    │
                    ├─→ [卖家修改价格]（可选，议价场景）
                    │       → 买家看到新价格后决定是否购买
                    │
                    └─→ [买家在聊天中点击"确认购买"] → 生成订单
                            │
                            │   订单状态：待面交
                            │   开始计时：72小时
                            │   同一商品同时只允许一个待面交订单（先到先得）
                            │
                            ├─→ [72小时内完成面交]
                            │       │
                            │       └─→ [买家点击"确认收货"]
                            │               │
                            │               └─→ 订单状态：已完成
                            │                       │
                            │                       └─→ [双方互评]（7天窗口期）
                            │                               │
                            │                               ├─→ 双方都评价 → 订单状态：已评价
                            │                               └─→ 7天未评价 → 系统自动默认好评
                            │                                       → 订单状态：已评价
                            │                                       → 商品自动下架
                            │
                            ├─→ [72小时超时] → 订单自动取消 → 通知双方 → 商品恢复在售
                            │
                            ├─→ [买家取消交易] → 选择取消原因 → 订单取消 → 通知卖家 → 商品恢复在售
                            │
                            └─→ [卖家取消交易] → 选择取消原因 → 订单取消 → 通知买家 → 商品恢复在售

        * 买家确认收货后7天未操作 → 系统自动确认收货
```

### 4.2 各功能详细说明

#### 4.2.1 登录注册模块 `MODULE-USER`

**微信登录：**
- 用户点击「微信登录」按钮
- 调用 `uni.login()` 获取code
- 后端用code调用微信API换取openId和session_key
- 若openId已存在则直接登录，否则自动创建新用户
- 登录成功后生成JWT Token返回
- 首次登录需弹窗展示并勾选同意《用户协议》和《隐私政策》

**手机号+密码登录：**
- 输入手机号和密码
- 后端校验账号密码（BCrypt验证）
- 登录成功后生成JWT Token返回

**短信验证登录：**
- 输入手机号，点击「获取验证码」
- 后端生成6位数字验证码，有效期5分钟，存Redis
- 同一手机号60秒内不可重复发送，每天上限10条
- 用户输入验证码，后端校验
- 未注册手机号自动创建账号

**用户协议与隐私政策：**
- 登录页底部展示「登录即代表同意《用户协议》和《隐私政策》」
- 《用户协议》和《隐私政策》为可点击链接，跳转至协议详情页
- 协议详情页为富文本展示（内容后台可维护或前端静态页面）
- 首次登录必须勾选同意才可继续，否则提示「请先同意用户协议」
- 协议内容应包含：用户信息收集说明、交易规则、免责声明、违规处罚规则

#### 4.2.2 校园认证模块 `MODULE-AUTH`

**认证信息：**
- 学院（下拉选择，数据来源：学院表，后台可配置）
- 学号（手动输入，校验格式：纯数字，长度限制）
- 班级（手动输入）
- 认证材料（上传一卡通照片 或 3.0系统截图，仅支持JPG/PNG/JPEG，最大5MB）

**认证状态：**

| 状态值 | 状态名 | 说明 |
|--------|--------|------|
| 0 | 未认证 | 默认状态 |
| 1 | 审核中 | 已提交待审核 |
| 2 | 已认证 | 审核通过 |
| 3 | 已驳回 | 审核不通过，可重新提交 |

**审核规则：**
- 管理员在后台查看认证申请
- 核实一卡通/3.0截图中的学号与填写信息是否一致
- 通过则标记已认证，驳回需填写驳回原因
- 同一学号只能被一个账号认证，防止冒用

#### 4.2.3 首页模块 `MODULE-HOME`

**校区切换：**
- 顶部左侧显示当前校区名称（如「广轻大 南海校区」），点击展开校区选择器
- 校区列表来源：campus表，后台可动态增删
- 切换校区后，商品列表和Banner按该校区筛选
- 用户选择的校区存储在本地（uni.setStorageSync），下次打开自动记忆

**Banner运营位：**
- 顶部轮播Banner，自动轮播间隔5秒，支持手动滑动
- 每条Banner包含：图片、标题、跳转类型（商品详情/活动页/外部链接）、跳转地址、排序、上下架状态、展示校区、有效期（开始时间/结束时间）
- 后台可配置，支持按校区投放不同Banner

**分类入口：**
- 图标+文字形式横向展示一级分类
- V1.0分类：书籍、服饰、生活、电子设备、运动设备、潮玩娱乐
- 超过一行时显示「更多」入口，点击展开全部分类
- 点击分类跳转至搜索结果页（预设该分类筛选条件）

**最新发布商品流：**
- 列表形式展示最新发布的商品（status=在售）
- 默认展示当前选中校区的商品
- 每个商品卡片显示：首图、标题、磨损程度标签、校区、价格、发布时间
- 支持下拉刷新、上拉加载更多
- 每页加载20条

#### 4.2.4 搜索筛选模块 `MODULE-SEARCH`

**搜索功能：**
- 首页顶部搜索框，点击进入搜索页
- 搜索页展示：搜索历史 + 热门搜索
- 搜索历史：本地存储（uni.setStorageSync），最多20条，按时间倒序，支持单条删除和一键清空
- 热门搜索：来源search_keyword表（is_hot=1），后台可配置 + 根据搜索频次自动统计
- 输入关键词后点击搜索或键盘回车，跳转搜索结果页
- 搜索时记录关键词到search_keyword表（search_count+1）

**筛选条件：**

| 筛选项 | 选项值 | 默认值 |
|--------|--------|--------|
| 分类 | 全部分类 / 各一级分类 | 全部分类 |
| 校区 | 全部校区 / 南海北 / 南海南 / 新港 | 当前选中校区 |
| 价格 | 价格不限 / 自定义区间（最低价-最高价） | 价格不限 |
| 排序 | 最新发布 / 价格从低到高 / 价格从高到低 | 最新发布 |

**搜索结果：**
- 顶部显示结果总数（如「搜索结果 128件」）
- 列表形式展示，每条显示：首图、标题、磨损程度、价格、校区
- 支持上拉加载更多
- 无结果时展示空状态组件（图片+文案「暂无相关商品」）

#### 4.2.5 商品详情模块 `MODULE-PRODUCT`

**商品信息展示：**
- 顶部图片轮播（支持左右滑动，点击单张图片全屏预览，支持缩放）
- 价格区域：二手价格（大字红色）、原价（删除线灰色）、磨损程度标签
- 商品描述（多行文本展示）
- 商品标签（分类标签，如「电子产品」「平板」「Apple」）
- 数据展示：浏览数、收藏数、发布时间
- 面交信息：交易校区、面交地点

**卖家信息区域：**
- 头像、昵称、认证状态标识（已认证/未认证）
- 用户综合评分
- 点击卖家区域跳转至**卖家个人主页**

**顶部右上角操作菜单「⋮」：**
- 如果是**自己的商品**：编辑、下架/上架、删除
- 如果是**别人的商品**：分享、举报

**底部固定操作栏：**
- ☆ 收藏 / 取消收藏（带收藏数）
- ⚠ 举报（跳转举报页）
- 「我想要」按钮（跳转IM聊天页）

**微信分享：**
- 商品详情页支持转发给微信好友
- 分享卡片：商品首图 + 标题 + 价格
- 分享路径：`/pages/product/detail/detail?id={productId}`

#### 4.2.9 订单管理模块 `MODULE-ORDER`

**订单状态流转：**


| 状态值 | 状态名 | 买家可见操作 | 卖家可见操作 |

| 5 | 已取消 | 删除订单 | 删除订单 |

- 格式：`TD{yyyyMMddHHmmss}{4位随机数}`

**订单列表：**
- Tab切换：「我买到的」/「我卖出的」

- 操作按钮根据状态和角色动态显示
- 支持上拉加载更多

**取消规则：**
- 买家和卖家均可取消
- 取消时需选择取消原因：
  - 双方协商取消
  - 对方无响应
  - 不想买了/不想卖了
  - 其他

- 订单取消后通知对方（站内通知）

- 订单创建时设置 `expire_time = create_time + 72小时`

- 自动取消时 `cancel_by = 0`（系统）

#### 4.2.10 评价系统模块 `MODULE-REVIEW`

**评价维度：**

| 维度 | 类型 | 范围 | 说明 |
|------|------|------|------|
| 商品描述相符 | 星级 | 1-5 | 商品与描述是否一致 |
| 沟通态度 | 星级 | 1-5 | 对方沟通是否友好 |
| 交易体验 | 星级 | 1-5 | 整体交易满意度 |
| 文字评价 | 文本 | 0-200字 | 可选 |

**评分计算：**
- 单次评价得分 = (商品描述相符 + 沟通态度 + 交易体验) / 3
- 用户综合评分 = 所有收到的评价的单次评价得分的平均值
- 保留一位小数
- 新用户默认评分5.0
- 每次收到新评价时重新计算综合评分并更新user表score字段

**评价规则：**
- 交易完成（已完成状态）后双方均可评价
- 评价窗口期：交易完成后7天
- 超过7天未评价，系统自动给予默认好评（三维度均为5分，is_auto=1）
- 评价提交后不可修改、不可删除
- 双方评价互相不可见（在都评价完或窗口期结束后才可见，防止报复性差评）

### 5.2 数据库表设计

#### 5.2.1 用户表（user）`MODULE-USER`

```sql
CREATE TABLE IF NOT EXISTS `user` (
  `id` bigint(20) NOT NULL AUTO_INCREMENT COMMENT '主键',
  `open_id` varchar(64) COLLATE utf8mb4_unicode_ci DEFAULT NULL COMMENT '微信openid',
```

## 七、异常流程与边界情况处理 【全局】

### 7.1 交易异常

| 场景 | 处理方式 |
|------|----------|
| 商品在买家确认购买前被其他买家成交 | 商品详情页 `hasActiveOrder=true`，「我想要」仍可点击但「确认购买」提示「该商品已有待交易订单，请稍后再试」 |
| 买家确认购买后商品被卖家修改价格 | 已有订单价格不变，新价格仅影响后续新订单 |
| 买家确认购买后卖家下架商品 | 在售且有待面交订单时禁止下架，提示「请先完成或取消进行中的交易」 |
| 72小时超时自动取消 | 定时任务扫描，批量取消并通知双方，商品恢复在售 |
| 7天自动确认收货 | 定时任务扫描，自动确认并通知双方 |
| 买家/卖家账号被封禁 | 进行中的订单标记「已取消」，cancel_by=0（系统），通知对方 |
| 同一商品多个买家同时「确认购买」 | 数据库层面乐观锁，先到先得，后者提示「该商品已有待交易订单」 |
| 订单取消后买家再次购买同一商品 | 允许，生成新订单 |
| 买家购买自己的商品 | 前后端均校验，提示「不能购买自己的商品」 |

### 7.2 商品异常

| 场景 | 处理方式 |
|------|----------|
| 发布商品图片上传失败 | 提示「图片上传失败，请重试」 |
| 单张图片超过5MB | 前端拦截提示「图片大小不能超过5MB」 |
| 图片格式不支持 | 前端拦截提示「仅支持JPG、PNG、JPEG格式」 |
| 商品审核驳回后重新编辑 | 状态重置为待审核，重新进入审核队列 |
| 90天自动下架 | 定时任务扫描，自动下架并通知卖家 |
| 违禁商品举报成立 | 管理员强制下架，通知卖家 |
| 商品删除时有进行中的订单 | 禁止删除，提示「请先完成或取消进行中的交易」 |

### 7.3 用户异常

| 场景 | 处理方式 |
|------|----------|
| 验证码过期 | 提示「验证码已过期，请重新获取」 |
| 60秒内重复发送验证码 | 前端倒计时 + 后端Redis校验，提示「请60秒后重试」 |
| 每日验证码超限（10条） | 提示「今日验证码发送次数已达上限」 |
| 同一学号重复认证 | 提示「该学号已被其他账号认证」 |
| 认证材料模糊不清 | 管理员驳回，原因「认证材料不清晰，请重新上传」 |
| 账号注销期间登录 | 提示「账号注销中，是否恢复账号？」提供恢复操作 |
| 封禁用户尝试操作 | 返回特定错误码，前端提示「账号已被封禁，原因：{原因}」 |
| Token过期 | 返回code=401，前端跳转登录页 |
| 密码错误超过5次 | 账号锁定15分钟 |

### 7.4 系统异常

| 场景 | 处理方式 |
|------|----------|
| 服务器异常（500） | 前端统一提示「服务器繁忙，请稍后重试」 |
| 网络超时 | 前端提示「网络连接超时，请检查网络」|
| OpenIM连接断开 | 自动重连机制，最多重试3次，失败提示用户 |
| 文件存储空间不足 | 监控告警，管理员扩容 |
| 并发操作同一商品 | 数据库乐观锁/Redis分布式锁保证一致性 |
| 重复提交（表单/订单） | 前端按钮防重复点击 + 后端幂等性校验 |

### 9.3 定时任务设计

| 任务 | Cron | 逻辑 |
|------|------|------|
| 订单超时取消 | `0 */5 * * * ?` 每5分钟 | expire_time < NOW() 且 status=1 → status=5，cancel_by=0 |
| 自动确认收货 | `0 0 2 * * ?` 每天2点 | confirm_deadline < NOW() 且 status=1 → status=3 |
| 自动好评 | `0 0 3 * * ?` 每天3点 | complete_time + 7天 < NOW() 且 status=3 → 生成默认评价 → status=4 |
| 商品自动下架 | `0 0 4 * * ?` 每天4点 | auto_off_time < NOW() 且 status=1 → status=2 |
| 账号注销清理 | `0 0 5 * * ?` 每天5点 | deactivate_time + 30天 < NOW() 且 status=2 → 永久删除 |
