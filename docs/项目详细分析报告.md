# 校园二手交易平台 - 项目详细分析报告

## 一、项目整体架构概述

本项目是一个功能完整的**校园二手物品交易平台**，采用典型的**前后端分离架构**设计，整体技术栈如下：

### 1.1 技术架构总览

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **后端** | Spring Boot 3.3.7 | Java 17 主流微服务框架 |
| **数据库** | MySQL 8.0 | 关系型数据存储 |
| **缓存** | Redis | Session缓存、消息队列、热点数据缓存 |
| **ORM** | MyBatis Plus 3.5.7 | 简化数据库操作 |
| **认证** | JWT (jjwt 0.12.5) | 无状态令牌认证 |
| **WebSocket** | Spring WebSocket | 实时聊天功能 |
| **定时任务** | Spring Schedule | 订单超时处理等 |
| **小程序端** | UniApp + Vue 3 | 跨平台小程序框架 |
| **管理后台** | Vue 3 + Vite + Axios | 现代前端工程化 |

### 2.2 核心数据表详解

#### 2.2.1 用户表（user）

```sql
CREATE TABLE `user` (
  `id` bigint(20) PRIMARY KEY AUTO_INCREMENT,
  `open_id` varchar(64)        -- 微信openid（微信登录用）
  `nick_name` varchar(32)      -- 昵称
  `username` varchar(50)       -- 用户名（手机号）
  `password` varchar(255)      -- 密码（BCrypt加密）
  `avatar_url` varchar(255)    -- 头像URL
  `phone` varchar(11)          -- 手机号
  `campus_id` bigint(20)       -- 所属校区ID
  `auth_status` tinyint(4)    -- 认证状态：0-未认证 1-审核中 2-已认证 3-已驳回
  `score` decimal(3,1)         -- 用户评分（1-5分）
  `status` tinyint(4)          -- 账号状态：0-封禁 1-正常 2-注销中
  `ban_reason` varchar(255)    -- 封禁原因
  `deactivate_time` datetime  -- 注销申请时间
  `last_login_time` datetime   -- 最后登录时间
  -- 时间戳字段
)
```

**关键索引**：`idx_open_id`（微信登录）、`idx_phone`（手机号）、`idx_campus_id`、`idx_status`

#### 2.2.2 商品表（product）

```sql
CREATE TABLE `product` (
  `id` bigint(20) PRIMARY KEY AUTO_INCREMENT,
  `user_id` bigint(20)         -- 发布者ID
  `title` varchar(50)          -- 商品标题
  `description` varchar(500)  -- 商品描述
  `price` decimal(10,2)       -- 二手价格
  `original_price` decimal(10,2) -- 原价
  `category_id` bigint(20)    -- 分类ID
  `condition_level` tinyint(4) -- 成色：1-全新 2-99新 3-9成新 4-8成新 5-7成新及以下
  `campus_id` bigint(20)       -- 交易校区
  `meeting_point_id` bigint(20) -- 面交地点ID
  `meeting_point_text` varchar(100) -- 自定义面交地点
  `images` varchar(2000)       -- 商品图片JSON数组
  `view_count` int(11)         -- 浏览量
  `favorite_count` int(11)     -- 收藏量
  `status` tinyint(4)          -- 状态：0-待审核 1-在售 2-已下架 3-已售出 4-审核驳回
  `reject_reason` varchar(255) -- 驳回原因
  `review_time` datetime       -- 审核时间
  `reviewer_id` bigint(20)    -- 审核人ID
  `auto_off_time` datetime    -- 自动下架时间（发布后90天）
  -- 时间戳字段
)
```

**关键索引**：联合索引 `idx_status_campus_category_create` 用于商品列表查询

#### 2.2.3 订单表（trade_order）

```sql
CREATE TABLE `trade_order` (
  `id` bigint(20) PRIMARY KEY AUTO_INCREMENT,
  `order_no` varchar(32)      -- 订单号（唯一）
  `product_id` bigint(20)      -- 商品ID
  `buyer_id` bigint(20)        -- 买家ID
  `seller_id` bigint(20)       -- 卖家ID
  `price` decimal(10,2)        -- 成交价格
  `campus_id` bigint(20)       -- 面交校区
  `meeting_point` varchar(100) -- 面交地点
  `status` tinyint(4)          -- 状态：1-待面交 3-已完成 4-已评价 5-已取消
  `cancel_reason` varchar(255) -- 取消原因
  `cancel_by` bigint(20)       -- 取消操作人（0表示系统）
  `expire_time` datetime       -- 订单过期时间（创建后72小时）
  `confirm_deadline` datetime  -- 自动确认截止时间（创建后7天）
  `complete_time` datetime    -- 完成时间
  `is_deleted_buyer` tinyint   -- 买家是否删除
  `is_deleted_seller` tinyint  -- 卖家是否删除
  -- 时间戳字段
)
```

#### 2.2.4 会话表（chat_session）- 重要设计说明

```sql
CREATE TABLE `chat_session` (
  `id` bigint(20) PRIMARY KEY AUTO_INCREMENT,
  `user_id` bigint(20)        -- 当前用户ID
  `peer_id` bigint(20)        -- 对话方用户ID
  `product_id` bigint(20)     -- 关联商品ID（可选）
  `last_msg` varchar(255)     -- 最后一条消息摘要
  `last_msg_type` tinyint(4)  -- 最后消息类型：1-文本 2-商品卡片 3-订单卡片 4-系统
  `unread` int(11)            -- 未读数
  `last_time` datetime        -- 最后消息时间
  `is_top` tinyint(1)         -- 是否置顶
  `is_deleted` tinyint(1)     -- 是否删除
  -- 时间戳字段
)
```

**设计说明**：聊天消息内容由OpenIM第三方服务管理，本地仅存储会话关联关系和未读计数。

### 3.2 核心模块详解

#### 3.2.1 用户模块（MODULE-USER）

**功能定位**：整个系统的基础模块，负责用户注册、登录、信息管理

**核心功能**：
- **微信登录**：通过微信openId进行小程序授权登录
- **手机号登录**：短信验证码登录
- **账号密码登录**：用户名+密码登录（BCrypt加密存储）
- **用户信息管理**：修改昵称、头像、性别、手机号
- **用户封禁/解封**：管理员操作

**关键API**：

```
MiniUserController:
  GET  /mini/user/info           - 获取当前用户信息
  POST /mini/user/update         - 更新用户信息
  POST /mini/user/change-phone   - 更换手机号
  GET  /mini/user/stats          - 获取用户统计数据

AdminUserController:
  GET  /admin/user/page         - 分页查询用户列表
  GET  /admin/user/{id}          - 获取用户详情
  POST /admin/user/ban           - 封禁用户
  POST /admin/user/unban         - 解封用户
```

**关键Service**：
- `UserService`：用户CRUD、登录验证、用户统计
- `UserServiceImpl`：实现具体的业务逻辑

**关键Entity**：`User.java`（用户实体，包含26个字段）

**模块联动**：
- 上游：认证模块（依赖用户认证状态）
- 下游：商品模块（发布者）、订单模块（买家/卖家）、消息通知模块

---

#### 3.2.2 商品模块（MODULE-PRODUCT）

**功能定位**：商品发布、浏览、搜索、管理的核心模块

**核心功能**：
- **商品发布**：发布二手商品，包含标题、描述、价格、成色、图片等
- **商品编辑**：修改商品信息（仅发布者可操作）
- **商品删除**：软删除商品
- **商品审核**：待审核 → 已通过 / 已驳回（管理员操作）
- **商品搜索**：关键词搜索 + 分类筛选 + 价格区间 + 校区筛选
- **商品排序**：最新发布、价格从低到高、价格从高到低
- **浏览量统计**：每次查看详情+1

**关键API**：

```
MiniProductController:
  POST /mini/product/publish     - 发布商品
  POST /mini/product/update      - 编辑商品
  POST /mini/product/update-price - 修改价格
  POST /mini/product/off-shelf   - 下架商品
  POST /mini/product/on-shelf    - 上架商品
  POST /mini/product/delete      - 删除商品
  GET  /mini/product/detail/{id} - 商品详情
  GET  /mini/product/list        - 商品列表（分页+筛选+排序）
  GET  /mini/product/my-list     - 我的发布列表

AdminProductController:
  GET  /admin/product/page      - 管理员商品列表
  GET  /admin/product/{id}       - 管理员商品详情
  POST /admin/product/approve    - 审核通过
  POST /admin/product/reject     - 审核驳回
  POST /admin/product/batch-approve - 批量审核通过
  POST /admin/product/force-off  - 强制下架
```

**关键Service**：
- `ProductService`：商品CRUD、上下架、审核、列表查询
- `ProductAsyncService`：异步处理（如发布成功后的通知）

**关键Entity**：`Product.java`（商品实体，24个字段）

**关键DTO**：
- `ProductPublishDTO`：商品发布数据传输对象
- `ProductUpdateDTO`：商品更新数据传输对象

**关键VO**：
- `ProductListVO`：商品列表视图对象
- `ProductDetailVO`：商品详情视图对象

**模块联动**：
- 上游：用户模块（发布者）、分类模块、校区模块、面交地点模块
- 下游：订单模块（基于商品创建订单）、收藏模块、搜索模块、评价模块（通过订单）、聊天模块（关联商品）

#### 3.2.3 交易订单模块（MODULE-ORDER）

**功能定位**：买卖双方交易流程管理的核心模块

**核心功能**：
- **创建订单**：基于商品创建交易订单
- **订单列表**：买家订单列表、卖家订单列表
- **订单详情**：查看订单详细信息
- **取消订单**：买家/卖家/系统取消
- **确认收货**：交易完成后确认收货
- **订单状态流转**：待面交 → 已完成 → 已评价

**订单状态机**：

```
[待面交(1)] ──取消──> [已取消(5)]
    │
    └──确认收货──> [已完成(3)]
                       │
                       └──评价──> [已评价(4)]

系统自动：
  [待面交] ──超时72小时──> [已取消(5)]
  [待面交] ──7天未确认──> [已完成(3)]（自动确认）
```

**关键API**：

```
MiniOrderController:
  POST /mini/order/create        - 创建订单
  GET  /mini/order/list          - 订单列表
  GET  /mini/order/detail/{id}   - 订单详情
  POST /mini/order/cancel        - 取消订单
  POST /mini/order/confirm      - 确认收货

AdminOrderController:
  GET  /admin/order/page         - 订单列表（管理）
  GET  /admin/order/{id}         - 订单详情（管理）
```

**关键Service**：
- `TradeOrderService`：订单CRUD、状态管理

**关键Entity**：`TradeOrder.java`（订单实体，22个字段）

**关键VO**：
- `OrderListVO`：订单列表视图
- `OrderDetailVO`：订单详情视图

**模块联动**：
- 上游：商品模块（基于商品创建）、用户模块（买家/卖家）
- 下游：评价模块（交易完成后可评价）、消息通知模块（订单状态变更通知）

---

#### 3.2.4 评价模块（MODULE-REVIEW）

**功能定位**：交易完成后的信用评价体系

**核心功能**：
- **提交评价**：交易完成后对对方进行评价
- **评价内容**：商品描述相符、沟通态度、交易体验（各1-5分）
- **自动评价**：超过一定时间未手动评价，自动生成好评
- **查看评价**：查看用户的所有评价

**关键API**：

```
MiniReviewController:
  POST /mini/review/submit      - 提交评价
  GET  /mini/review/list        - 评价列表
  GET  /mini/review/detail/{id}  - 评价详情
```

**关键Service**：
- `ReviewService`：评价CRUD、统计

**关键Entity**：`Review.java`（评价实体，包含3个评分维度）

**模块联动**：
- 上游：订单模块（基于订单创建评价）
- 下游：用户模块（影响用户评分score）

---

#### 3.2.5 聊天模块（MODULE-CHAT）

**功能定位**：买卖双方实时沟通功能

**技术实现**：
- **WebSocket通信**：基于Spring WebSocket实现
- **消息分发**：MessageDispatcher根据消息类型分发处理
- **会话管理**：维护用户会话列表和未读计数
- **消息存储**：本地存储会话元数据，消息内容由OpenIM第三方服务管理

**WebSocket架构**：

```
┌─────────────────────────────────────────────────────────────┐
│                    WebSocket 处理流程                        │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  客户端 ──发送消息──> WebSocketServer                       │
│       │                  │                                   │
│       │              MessageDispatcher                      │
│       │                  │                                   │
│       │        ┌─────────┼─────────┐                        │
│       │        ▼         ▼         ▼                        │
│       │    PingHandler ChatHandler ReadHandler              │
│       │        │         │         │                        │
│       │        │         ▼         │                        │
```

### 3.3 定时任务模块（SCHEDULED TASKS）

系统通过定时任务实现自动化处理，共7个定时任务：

| 任务类 | 功能 | 执行策略 | 说明 |
|--------|------|----------|------|
| `OrderExpireTask` | 订单超时取消 | 每5分钟 | 检查超过72小时未面交的订单，自动取消 |
| `OrderAutoConfirmTask` | 订单自动确认 | 每天0点 | 检查超过7天未确认的订单，自动确认完成 |
| `ProductAutoOffTask` | 商品自动下架 | 每天1点 | 检查发布超过90天的商品，自动下架 |
| `UserDeactivateTask` | 用户自动注销 | 每天2点 | 检查申请注销超过7天的用户，执行注销 |
| `ReviewRemindTask` | 评价提醒 | 每天10点 | 提醒交易完成但未评价的用户 |
| `ReviewAutoTask` | 自动好评 | 每天12点 | 超过14天未评价的交易，自动生成好评 |
| `WebSocketHeartbeatTask` | WebSocket心跳 | 每分钟 | 清理失效的WebSocket连接 |

**定时任务配置示例**（OrderExpireTask）：

```java
@Scheduled(cron = "0 */5 * * * ?")  // 每5分钟执行
public void execute() {
    // 查询过期订单
    // 取消订单
    // 恢复商品状态为在售
    // 发送通知给买卖双方
}
```
