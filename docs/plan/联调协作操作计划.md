# 前后端联调协作操作计划

> 版本：V1.1 | 更新日期：2026-03-15
> 模式：Trae IDE（执行者） + Kiro IDE（监督者）双IDE协作

---

## 📋 阶段概览

| 阶段 | 内容 | 预计时长 |
|------|------|----------|
| 阶段一 | 环境准备 | 30分钟 |
| 阶段二 | 核心模块联调 | 4-6小时 |
| 阶段三 | 业务流程联调 | 2-3小时 |
| 阶段四 | 管理端联调 | 1-2小时 |
| 阶段五 | 问题修复与验收 | 1-2小时 |

## 🚀 阶段一：环境准备

### 执行者任务
| 序号 | 任务 | 状态 |
|------|------|------|
| 1 | 启动MySQL服务（端口3306） | ☐ |
| 2 | 启动Redis服务（端口6379） | ☐ |
| 3 | 初始化数据库：`mysql -u root -p secondhand < sql/init.sql` | ☐ |
| 4 | 启动后端服务：`mvn spring-boot:run`（端口8080） | ☐ |
| 5 | 修改 `miniapp/utils/request.js`：`USE_MOCK = false` | ☐ |
| 6 | 打开微信开发者工具，勾选"不校验合法域名" | ☐ |
| 7 | 验证后端接口可用：Postman测试登录接口 | ☐ |

### 监督者任务
| 序号 | 任务 | 状态 |
|------|------|------|
| 1 | 确认环境配置文档已阅读 | ☐ |
| 2 | 验证执行者提供的环境状态 | ☐ |

### 交付物
- [ ] 环境就绪确认
- [ ] 后端接口可访问

## 📊 协作流程

```
┌─────────────────────────────────────────────────────────────┐
│  执行者（Trae）                                              │
│  1. 读取联调检查清单                                        │
│  2. 环境准备确认                                            │
│  3. 执行联调测试                                            │
│  4. 问题定位与修复                                          │
│  5. 创建 .debug-ready 信号文件                              │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  信号文件：.debug-ready                                     │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  监督者（Kiro）                                              │
│  1. 读取问题报告                                            │
│  2. 环境验证                                                │
│  3. 功能审查                                                │
│  4. 代码质量审查                                            │
│  5. 创建 .review-passed 或 .review-rejected               │
└─────────────────────────────────────────────────────────────┘
                              ↓
         ┌───────────────────────┐
         │  .review-passed ✅     │ → 下一模块
         │  .review-rejected ❌  │ → 返回执行者修复
         └───────────────────────┘
```
