## 📋 联调任务接收

当你收到联调任务时，按照以下流程执行：

### 1. 读取任务要求
```bash
# 查看联调检查清单中的具体检查项
# 位置：docs/联调检查清单.md
```

### 2. 环境准备检查
在开始调试前，必须确认以下环境已就绪：

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 后端服务运行中 | ☐ | 端口8080 |
| MySQL服务运行中 | ☐ | 端口3306 |
| Redis服务运行中 | ☐ | 端口6379 |
| USE_MOCK = false | ☐ | miniapp/utils/request.js |
| 微信开发者工具打开 | ☐ | 已勾选不校验域名 |

## 📝 问题反馈规范

当遇到无法解决的问题时，必须提供以下信息：

### 必须包含
1. **问题现象**：具体描述（如：点击登录按钮返回"账号不存在"）
2. **操作步骤**：1.打开小程序 → 2.点击登录 → 3.输入手机号 → 4.点击登录
3. **预期结果**：正常登录成功，返回Token
4. **实际结果**：提示"账号不存在"
5. **相关代码**：涉及的文件和函数

### 推荐包含
6. **后端日志**：关键日志输出
7. **接口返回**：Postman测试结果
8. **数据库状态**：相关表数据

## ✅ 联调完成标准

每个模块联调完成后，必须确认：

| 标准 | 说明 |
|------|------|
| 功能可用 | 核心业务流程跑通 |
| 数据正确 | 数据库正确落库 |
| 异常处理 | 错误提示友好 |
| 日志可查 | 后端有相关日志 |

---

## ⚠️ 避免"AI打转"

### 常见导致打转的行为
- ❌ 只说"这里不对"，不说具体哪里不对
- ❌ 不提供日志和错误信息
- ❌ 不使用Postman独立验证接口
- ❌ 跳过环境检查直接改代码

### 正确做法
- ✅ 提供完整的错误信息和操作步骤
- ✅ 先用Postman验证后端接口可用
- ✅ 检查环境配置后再定位代码问题
- ✅ 记录每次尝试的结果

## 🎯 联调优先级

建议按以下顺序进行：

1. **登录模块** - 最基础，后续功能依赖
2. **首页/商品列表** - 验证数据展示
3. **商品详情/发布** - 验证CRUD
4. **IM即时通讯** - WebSocket连通
5. **订单流程** - 完整业务流程
6. **评价/消息中心** - 辅助功能
7. **后台管理** - 管理员功能
