### 问题1：订单创建接口 ✅ 已修复

**修复日期：** 2026-03-18

**问题描述：**
后端DTO要求4个必填字段，但前端只传了1个字段，会导致后端返回400错误。

**修复方案：** 采用方案A - 简化后端DTO

**修复内容：**
1. 后端 `OrderCreateDTO.java` 简化为只接收 `productId` 字段
2. 后端 `TradeOrderServiceImpl.java` 自动从商品信息获取：
   - `price` ← `product.getPrice()`
   - `campusId` ← `product.getCampusId()`
   - `meetingPoint` ← `product.getMeetingPointText()`
3. 前端无需修改

**修复后接口参数：**
```json
{
  "productId": 100
}
```

**优点：**
- 接口简洁，前端调用方便
- 防止前端篡改价格
- 面交地点继承商品预设，符合业务场景
