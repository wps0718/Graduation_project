**WebSocket地址**：`ws://localhost:8080/ws/chat?token={jwt_token}`

**消息协议**：
```json
// 发送文本消息
{"type": "CHAT", "data": {"receiverId": xxx, "productId": xxx, "msgType": 1, "content": "xxx"}}

{"type": "CHAT", "data": {"receiverId": xxx, "productId": xxx, "msgType": 2}}

{"type": "PING"}
```

**REST接口**：
- `GET /mini/chat/sessions` - 会话列表
- `POST /mini/chat/session/create` - 创建会话
- `POST /mini/chat/session/delete` - 删除会话
- `POST /mini/chat/session/top` - 置顶/取消置顶会话
- `GET /mini/chat/messages?sessionKey=xxx` - 消息历史
- `POST /mini/chat/read` - 标记已读 (body: `{sessionKey: xxx}`)
- `GET /mini/chat/unread-total` - 未读消息总数

**预期实现结果**：用户可以通过WebSocket实时发送文字消息、商品卡片、订单卡片，会话列表展示所有聊天记录，未读数实时更新。

### 6.2 WebSocket重连机制

```javascript
// WebSocket断开后重连逻辑（小程序端）
let reconnectCount = 0;
const maxReconnectCount = 5;

function onSocketClose() {
  if (reconnectCount < maxReconnectCount) {
    setTimeout(() => {
      connectWebSocket();
      reconnectCount++;
    }, 3000); // 3秒后重连
  }
}
```

# nginx.conf 配置示例

http {
    upstream backend {
        server 127.0.0.1:8080;
    }

    server {
        listen 80;
        server_name your-domain.com;

        # 静态资源
        location /uploads/ {
            alias /var/www/html/uploads/;
        }

        # API反向代理
        location / {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
        }

        # WebSocket反向代理
        location /ws/ {
            proxy_pass http://backend;
            proxy_http_version 1.1;

    }
}
